프롬프트 잘 쓰는 법 💬

"AI에게 애매하게 말하면 결과도 애매해집니다."

프롬프트란 AI에게 전달하는 지시나 질문을 말합니다. 같은 목표라도 어떻게 말하느냐에 따라 결과가 크게 달라집니다. 좋은 프롬프트를 쓰는 것은 연습할수록 자연스럽게 늘어납니다.

좋은 프롬프트의 4가지 요소​

1. 구체적으로 말하기​

AI는 모호한 표현을 싫어합니다. 원하는 것을 구체적으로 설명할수록 좋은 결과를 얻습니다.

나쁜 예:

버튼 만들어줘.

좋은 예:

파란색 배경에 흰색 글씨로 "저장" 이라고 적힌 버튼을 만들어줘.
클릭하면 콘솔에 "저장 완료"가 출력되도록 해줘.

2. 맥락 제공하기​

AI는 현재 상황을 모릅니다. 어떤 프로젝트인지, 어떤 코드가 이미 있는지 알려주세요.

나쁜 예:

삭제 기능 추가해줘.

좋은 예:

위에서 만든 할 일 목록 앱에 삭제 기능을 추가해줘.
각 항목 오른쪽에 빨간색 "X" 버튼이 생기고,
클릭하면 해당 항목이 목록에서 사라지도록 해줘.

3. 결과물 형태 명시하기​

어떤 형태로 결과를 받고 싶은지 알려주면 더 원하는 형태로 받을 수 있습니다.

나쁜 예:

로그인 폼 만들어줘.

좋은 예:

HTML 파일 하나로 동작하는 로그인 폼을 만들어줘.
- 이메일 입력 필드
- 비밀번호 입력 필드 (입력값이 별표로 가려짐)
- 로그인 버튼
- 외부 라이브러리 없이 순수 HTML/CSS/JavaScript만 사용

4. 제약 조건 추가하기​

원하지 않는 것도 함께 말해주면 더 정확한 결과를 얻습니다.

예시:

간단한 계산기를 만들어줘.
- 더하기, 빼기, 곱하기, 나누기만 있으면 돼
- React나 Vue 같은 프레임워크 사용하지 말고
- 파일 하나로 만들어줘
- 디자인은 간단하게

나쁜 프롬프트 vs 좋은 프롬프트 비교​

예시 1: 에러 해결​

나쁜 예:

에러가 났어. 고쳐줘.

좋은 예:

아래 에러가 발생했어. 원인이 뭔지 설명해주고 수정된 코드를 보여줘.

에러 메시지:
TypeError: Cannot read properties of null (reading 'addEventListener')
    at script.js:15

관련 코드:
const button = document.getElementById('myBtn');
button.addEventListener('click', handleClick);

예시 2: 기능 추가​

나쁜 예:

검색 기능 넣어줘.

좋은 예:

현재 할 일 목록 앱에 검색 기능을 추가해줘.

- 목록 상단에 검색 입력창 추가
- 입력할 때마다 실시간으로 목록이 필터링됨
- 검색어를 지우면 전체 목록이 다시 보임
- 대소문자 구분 없이 검색

예시 3: 코드 설명 요청​

나쁜 예:

이 코드가 뭐야?

좋은 예:

아래 코드가 어떻게 동작하는지 단계별로 설명해줘.
초보자도 이해할 수 있게 쉽게 설명해줘.

[코드 붙여넣기]

바로 복사해서 사용할 수 있는 템플릿들입니다.

AI와 대화할 때 꿀팁​

한 번에 너무 많이 요청하지 마세요​

한 번의 요청으로 너무 많은 것을 요청하면 AI가 모든 것을 완벽하게 처리하기 어렵습니다.

이렇게 나눠서 요청해봅시다:

1단계: 기본 틀 만들어줘 2단계: 로그인 기능 추가해줘 3단계: 스타일 다듬어줘 4단계: 모바일에서도 잘 보이게 해줘

마음에 안 들면 바로 말해주세요​

AI는 피드백을 좋아합니다. 마음에 들지 않으면 솔직하게 말해주세요:

이거 마음에 안 들어. 버튼이 너무 커. 절반 크기로 줄여줘.
색상이 너무 화려해. 더 차분한 톤으로 바꿔줘.

"왜?"라고 물어보세요​

코드를 이해하고 싶을 때는 설명을 요청하세요:

방금 만든 코드에서 addEventListener가 뭐 하는 건지 설명해줘.
이 코드에서 forEach 부분이 어떻게 동작하는 건지 초보자에게 설명하듯 알려줘.

이렇게 하면 자연스럽게 코딩을 배울 수 있습니다.

이전 대화 맥락을 활용하세요​

같은 채팅 창에서 계속 대화하면 AI가 이전 내용을 기억합니다:

아까 만든 할 일 앱에서 체크박스 부분만 수정해줘.
방금 추가한 검색 기능에서 한글 검색이 안 되는 문제가 있어. 고쳐줘.

프롬프트 연습 과제​

아래 상황에서 어떻게 프롬프트를 써야 할지 직접 작성해보세요.

과제 1: 할 일 앱에 "마감 날짜" 필드를 추가하고 싶습니다.

과제 2: 앱이 이상하게 동작합니다. AI에게 도움을 요청하고 싶습니다.

과제 3: 앱의 폰트를 바꾸고 싶습니다.

직접 써보고, AI에게 실제로 요청해봅시다!

요약​

좋은 프롬프트의 핵심:

  1. 구체적으로 - 애매한 표현 피하기
  2. 맥락 포함 - 현재 상황 알려주기
  3. 결과물 형태 명시 - 어떤 형태로 받고 싶은지
  4. 제약 조건 - 원하지 않는 것도 말하기
  5. 단계적으로 - 한 번에 너무 많이 요청하지 않기
