# 나만의 개발 루틴 🔁

> "매일 30분이 한 달에 한 번 3시간보다 강력합니다."

실력은 한 번에 쑥 늘지 않습니다. 꾸준한 습관이 쌓여서 성장합니다. 이 챕터에서는 AI와 함께 코딩하는 루틴을 만드는 방법을 알아봅니다.

---

## 왜 루틴이 중요한가요?

바이브 코딩을 처음 시작할 때 많은 분들이 이런 패턴을 겪습니다:

1. 처음 며칠은 신나서 열심히 합니다
2. 갑자기 막히거나 복잡해집니다
3. 며칠 쉬다가 다시 시작하기가 어렵습니다
4. 포기합니다

이를 방지하는 가장 좋은 방법은 **작고 지속 가능한 루틴**을 만드는 것입니다.

---

## 하루 30분 루틴

매일 딱 30분만 AI와 코딩해봅시다. 처음 1주일은 이 루틴을 따라해보세요:

### 1단계 (5분): 오늘 할 것 결정하기

어제 하던 것을 이어서 하거나, 새로운 작은 기능 하나를 정합니다.

**예시:**
- "오늘은 버튼 스타일을 바꿔볼거야"
- "오늘은 검색 기능 하나만 추가해볼거야"
- "오늘은 모바일에서도 잘 보이게 해볼거야"

작은 목표일수록 좋습니다. **한 번에 하나씩**입니다.

### 2단계 (20분): AI와 함께 만들기

목표를 정했으면 Cursor를 열고 AI에게 요청합니다.

오늘의 목표가 완성될 때까지 AI와 대화합니다. 잘 안 되면 다르게 설명해보고, 오류가 나면 AI에게 보여줍니다.

### 3단계 (5분): 오늘 한 것 기록하기

간단하게라도 메모해두세요:

```
2025-01-15
- 할 일 앱에 검색 기능 추가 완료
- 한글 검색이 처음엔 안 됐는데 toLowerCase()로 해결
- 다음엔 날짜 표시 기능 추가해볼 예정
```

이 기록이 쌓이면 나중에 큰 도움이 됩니다.

---

## 작은 목표로 쪼개는 법

처음부터 큰 것을 만들려고 하면 막히게 됩니다. 작게 쪼개는 연습을 해봅시다.

### 예시: 날씨 앱 만들기

**잘못된 접근:**
```
날씨 앱 전체를 다 만들어줘.
위치 검색, 현재 날씨, 5일 예보, 시간별 그래프,
배경 테마 자동 변경까지 다 포함해서.
```

**좋은 접근 (단계별):**

1주차: 기본 화면 만들기
```
도시 이름을 입력하면 해당 도시가 크게 표시되는
간단한 날씨 앱 틀을 만들어줘.
일단 실제 데이터 없이 임시 더미 데이터로.
```

2주차: 실제 데이터 연동
```
OpenWeatherMap API를 사용해서
실제 날씨 데이터를 가져와서 표시해줘.
API 키는 내가 직접 입력할게.
```

3주차: 디자인 개선
```
날씨에 따라 배경 색상이 바뀌도록 해줘.
맑은 날은 파란색, 흐린 날은 회색, 비 오는 날은 어두운 파란색.
```

이렇게 하면 매주 뭔가 완성된 것이 생기고, 포기하지 않게 됩니다.

## 막혔을 때 탈출하는 방법

코딩하다 막히는 것은 정상입니다. 막혔을 때 해결하는 방법들입니다.

### 방법 1: 더 작게 쪼개기

해결이 안 된다면 요청을 더 작은 단위로 쪼개보세요.

막힌 상황:
```
드래그 앤 드롭으로 할 일 순서 변경이 안 돼.
```

작게 쪼개기:
```
일단 드래그 앤 드롭은 빼고,
위/아래 버튼으로 순서를 바꾸는 방법으로 구현해줘.
```

### 방법 2: 처음부터 다시 만들기

가끔은 처음부터 다시 만드는 게 더 빠를 때가 있습니다:

```
지금 코드가 너무 복잡해진 것 같아.
같은 기능인데 더 단순하게 처음부터 다시 짜줘.
```

### 방법 3: 다른 방식으로 물어보기

같은 목표를 다르게 표현해봅니다:

원래 요청:
```
실시간 검색 기능 만들어줘.
```

다르게 표현:
```
사용자가 검색창에 타이핑할 때마다
목록에서 일치하는 항목만 보여주는 필터 기능을 만들어줘.
keyup 이벤트를 사용해서.
```

### 방법 4: 잠깐 쉬기

막혔을 때 계속 붙잡고 있으면 지칩니다. 10~15분 쉬고 다시 오면 새로운 아이디어가 떠오를 때가 많습니다.

### 방법 5: AI에게 방향 물어보기

```
[기능명]을 구현하려고 하는데,
어떤 방식으로 접근하는 게 좋을지 2~3가지 방법을 알려줘.
나는 초보자니까 가장 간단한 방법을 먼저 추천해줘.
```

## 1주일 루틴 예시

처음 1주일을 이렇게 계획해봅시다:

| 날 | 목표 | 예상 시간 |
|----|------|----------|
| **1일차** | 할 일 앱 기본 기능 완성 | 30분 |
| **2일차** | 디자인 개선 (색상, 폰트) | 30분 |
| **3일차** | 검색 기능 추가 | 30분 |
| **4일차** | 모바일 화면 최적화 | 30분 |
| **5일차** | 데이터 저장 기능 (localStorage) | 30분 |
| **6일차** | 새로운 앱 아이디어 구상 + 시작 | 30분 |
| **7일차** | 이번 주 만든 것 복습 + 다음 주 계획 | 30분 |

7일 후에는 할 일 앱이 완성되고, 새 프로젝트도 시작하게 됩니다!

## 좋은 습관 만들기

### 같은 시간, 같은 장소에서

루틴의 핵심은 **자동화**입니다. 매일 같은 시간, 같은 장소에서 하면 뇌가 자동으로 "코딩 모드"로 전환됩니다.

예: "저녁 식사 후 9시, 책상에서 30분"

### 완성에 집중하기

완벽한 코드보다 **동작하는 코드**가 더 중요합니다. 일단 작동하면 완성입니다. 나중에 AI에게 개선을 요청할 수 있습니다.

### 작은 성공을 축하하기

버튼 하나 만들었다고 대단한 거 아닌 것 같아도, 그것도 성공입니다. 작은 성공을 인식하고 스스로를 칭찬해주세요.

### 만든 것을 다른 사람에게 보여주기

친구나 가족에게 "내가 이거 만들었어!"라고 보여주세요. 반응이 좋으면 더 만들고 싶어집니다.

## 나만의 프로젝트 아이디어 찾기

매일 쓰는 것, 불편한 것을 생각해보세요:

- 자주 하는 반복 작업을 자동화할 수 없을까?
- 정보를 정리하는 더 좋은 방법은 없을까?
- 취미와 관련된 뭔가를 만들 수 있지 않을까?

**쉬운 아이디어들:**
- 독서 기록 앱 (읽은 책, 별점, 메모)
- 운동 기록 앱 (날짜, 종목, 시간)
- 링크 저장 앱 (나중에 읽을 링크 모음)
- 명언 랜덤 표시 앱
- 생일 알리미 (가족/친구 생일 관리)

이 중 하나를 골라서 만들어보세요!

---

## 요약

루틴의 핵심:

1. **매일 30분** - 길게 하려 하지 말고 짧게 꾸준히
2. **하루 목표 하나** - 작은 것 하나만 완성하기
3. **기록하기** - 무엇을 했는지 간단히 메모
4. **막히면 쪼개기** - 더 작은 단위로 나누기
5. **완성에 집중** - 완벽보다 동작하는 것
