# 2강. 거대 모델의 바다에서 나만의 무기 고르기 🧠

> 시장에는 수많은 모델이 있지만, 모든 모델이 내 업무에 정답은 아닙니다.
> 바이브 코딩의 핵심은 **적재적소에 가장 유능한 직원을 배치하는 안목**에서 시작됩니다.

:::note 2026년 에이전트 시대
AI는 이제 단순히 코드를 짜주는 것을 넘어, 스스로 브라우저를 켜고 파일을 수정하며 결과물을 배포까지 하는 **에이전트** 역할을 수행합니다. Cursor나 IDE 설정 시 아래 모델들을 상황에 맞게 선택하세요.
:::

---

## 1. 모델별 강점 비교: 어떤 직원을 채용할 것인가?

우리가 다루는 주요 LLM들은 각자 전공 분야가 다릅니다. 이 특성을 알아야 엉뚱한 직원에게 엉뚱한 일을 시키지 않습니다.

---

### (1) Anthropic Claude 4.6: "감성과 코딩 실력을 겸비한 시니어 개발자"

| | |
|---|---|
| **강점** | 인간의 의도('바이브') 파악 능력 최고, 압도적인 코딩 실력, 속도·비용·지능의 균형 |
| **전략적 활용** | 실제 어플리케이션 코드 구현, UI 제작, 자동화 코딩 도구(Claude Code 등)의 기본 엔진 |
| **바이브** | 코드를 짤 때 가장 인간답고 현대적인 방식을 선호합니다. 우리 바이브 코딩의 **주력 직원**입니다 |

**라인업:**
- **Claude 4.6 Sonnet** — 속도, 비용, 지능의 완벽한 조화. 대부분의 코딩 작업에 최적
- **Claude 4.6 Opus** — 극도로 복잡한 로직이나 100만 토큰 이상의 대규모 코드베이스 전체 분석 시 사용

---

### (2) OpenAI GPT-5.2: "냉철한 논리력을 가진 수석 전략가"

| | |
|---|---|
| **강점** | 범용성, 압도적인 추론 능력, 단계별 사고(Chain-of-Thought) |
| **전략적 활용** | 복잡한 알고리즘 설계, 보안 취약점 분석, 논리적 결함이 있는 버그 수정 |
| **바이브** | 코드를 짜기 전 '생각하는 단계'를 거쳐 논리적 오류를 잡아냅니다 |

**라인업:**
- **GPT-5.2 Thinking** — 복잡한 추론에 특화. 과거 o1 모델을 계승한 추론 전문 모델
- **OpenAI Operator (Agent)** — 화면을 보고 마우스·키보드를 직접 조작해 코딩 환경을 세팅하는 에이전트

---

### (3) Google Gemini 3.1 Pro: "눈과 귀가 밝은 멀티모달 전문가"

| | |
|---|---|
| **강점** | 멀티모달 처리, 초대형 컨텍스트 윈도우, 방대한 공식 문서 실시간 이해 |
| **전략적 활용** | 수 시간 분량의 영상 가이드 분석, 수천 페이지 공식 문서 학습, 최신 라이브러리 문법 질의 |
| **바이브** | 최신 라이브러리의 공식 문서를 통째로 학습시켜 가장 정확한 최신 문법을 알려줍니다. **'Gemini 3 Deep Think'** 모드는 복잡한 논리 구조 해결에 특히 유용 |

## 2. 로컬 모델(Local LLM) 맛보기: 나만의 비밀 요원

클라우드 모델(GPT, Claude 등)은 강력하지만, 기업의 기밀이나 개인적인 데이터를 다룰 때는 망설여질 수 있습니다. 이때 필요한 것이 **내 컴퓨터에 직접 설치하는 로컬 모델**입니다.

- **대표 주자 - Llama 3 (Meta)**: 오픈 소스 모델 중 가장 강력한 성능을 자랑합니다
- **설치 도구 - Ollama**: 복잡한 설정 없이 명령어 한 줄로 내 PC에서 AI를 돌릴 수 있게 해주는 도구입니다
- **활용의 가치**: 인터넷이 연결되지 않은 환경(망 분리 보안 환경)에서도 나만의 AI 비서를 가질 수 있습니다

:::info 로컬 모델의 실전 가치
이는 이후 강의에서 다룰 하이닉스 실무 환경에서 매우 중요한 무기가 됩니다.
보안이 중요한 기업 환경일수록 로컬 모델의 전략적 가치가 높아집니다.
:::

## 3. 실전: API 환경 설정 (나의 직원들과 계약하기)

에이전트를 만들기 위해서는 채팅창을 넘어 **API(Application Programming Interface)** 를 통해 대화해야 합니다.

### API Key 발급

각 플랫폼에서 고유한 키를 발급받습니다:

| 모델 | 발급 위치 |
|------|----------|
| GPT-4o | platform.openai.com |
| Claude | console.anthropic.com |
| Gemini | aistudio.google.com |

### 환경 변수 관리

이 키들은 **절대 코드에 직접 노출하지 않고** `.env` 파일에 보관합니다.

```bash
# .env 파일 예시
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
GOOGLE_API_KEY=AIza...
```

### 연결 확인

n8n이나 Cursor에서 이 키를 입력하고 첫 번째 "Hello AI" 응답을 받아보는 순간, 여러분은 **전 세계에서 가장 유능한 직원들과 계약을 체결**한 것입니다.

## 💡 바이브 포인트: 상황에 맞는 최고의 직원을 채용하는 안목

> "좋은 리더는 모든 일을 잘하는 사람이 아니라, 그 일을 가장 잘할 사람을 알아보는 사람입니다."

바이브 코딩은 무조건 비싼 모델을 쓰는 것이 아닙니다:

- **단순한 데이터 분류** → Gemini Flash나 GPT-4o mini 같은 '가벼운 인턴'에게 맡겨 비용을 아끼고
- **아키텍처 설계 같은 핵심 업무** → Claude 3.5 Sonnet 같은 '에이스 부장'에게 맡기는 것

이것이 바로 경량문명 시대의 리더가 갖춰야 할 **'지능 운용 능력'** 입니다.

:::tip 모델 선택 요약
| 업무 유형 | 추천 모델 |
|----------|----------|
| 전략 수립 / 복잡한 추론 | GPT-4o |
| 코드 작성 / 감성적 글쓰기 | Claude 3.5 Sonnet |
| 영상/이미지 분석 / 대용량 문서 | Gemini 1.5 Pro |
| 단순 반복 / 비용 절감 | GPT-4o mini / Gemini Flash |
| 보안 환경 / 오프라인 | Llama 3 (Ollama) |
:::
